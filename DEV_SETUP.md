# MicroManus — Local Development Setup

## Prerequisites
- Node.js 18+
- npm 9+
- A Supabase project (free tier is fine)
- A Stripe account (test mode)
- A Tavily API key (free tier: 1,000 credits/month)
- Stripe CLI (for local webhook forwarding)

---

## Step 1 — Install dependencies

```bash
npm install
```

---

## Step 2 — Configure environment variables

Copy the example file and fill in your values:

```bash
cp .env.local.example .env.local
```

Then edit `.env.local`:

| Variable | How to get it |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase Dashboard → Project → Settings → API → Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase Dashboard → Project → Settings → API → anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase Dashboard → Project → Settings → API → service_role key |
| `STRIPE_SECRET_KEY` | Stripe Dashboard → Developers → API keys → Secret key (test mode: `sk_test_...`) |
| `STRIPE_WEBHOOK_SECRET` | Generated by Stripe CLI in Step 5 below |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | Stripe Dashboard → Developers → API keys → Publishable key |
| `TAVILY_API_KEY` | https://app.tavily.com → API Keys |
| `BRAVE_SEARCH_API_KEY` | (Optional) https://api.search.brave.com → My plan |
| `ENCRYPTION_SECRET` | Generate with: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` |
| `NEXT_PUBLIC_SITE_URL` | `http://localhost:3000` for local; your Vercel URL for production |

## Step 3 — Set up the Supabase database

1. Go to your Supabase project → SQL Editor
2. Copy and paste the contents of **`supabase/migrations/001_initial_schema.sql`** and run it
3. Copy and paste the contents of **`supabase/migrations/002_rpc_decrement_credits.sql`** and run it

---

## Step 4 — Configure OAuth providers in Supabase

### Google OAuth
1. Go to [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → Credentials
2. Create OAuth 2.0 Client ID → Web application
3. Add Authorized redirect URI: `https://<your-project-id>.supabase.co/auth/v1/callback`
4. In Supabase Dashboard → Authentication → Providers → Google → enable and paste Client ID + Secret

### GitHub OAuth
1. Go to GitHub Settings → Developer settings → OAuth Apps → New OAuth App
2. Homepage URL: `http://localhost:3000` (update to your domain in production)
3. Authorization callback URL: `https://<your-project-id>.supabase.co/auth/v1/callback`
4. In Supabase Dashboard → Authentication → Providers → GitHub → enable and paste Client ID + Secret

### Redirect URLs (Supabase Auth)
In Supabase Dashboard → Authentication → URL Configuration:
- Add `http://localhost:3000/**` to the Redirect URLs allowlist
- Add your production URL `https://your-domain.vercel.app/**` too

---

## Step 5 — Set up Stripe CLI for local webhooks

Install Stripe CLI: https://stripe.com/docs/stripe-cli#install

Then run:
```bash
stripe listen --forward-to localhost:3000/api/stripe/webhook
```

The CLI will print a webhook signing secret (starts with `whsec_...`). Copy it to:
```
STRIPE_WEBHOOK_SECRET=whsec_...
```

Restart `npm run dev` after updating `.env.local`.

---

## Step 6 — Run locally

```bash
npm run dev
```

Visit `http://localhost:3000`

---

## Deployment to Vercel

1. Push your repo to GitHub
2. Import the project in Vercel
3. Add all environment variables from `.env.local` in the Vercel dashboard (Settings → Environment Variables)
4. For `NEXT_PUBLIC_SITE_URL`, set it to your Vercel production URL (e.g., `https://micromanus.vercel.app`)
5. In Stripe Dashboard → Webhooks → Add endpoint: `https://your-domain.vercel.app/api/stripe/webhook`
   - Select event: `checkout.session.completed`
   - Copy the signing secret to `STRIPE_WEBHOOK_SECRET` in Vercel env vars
6. Update Google/GitHub OAuth redirect URLs to include your Vercel domain
7. Update Supabase Auth redirect URLs allowlist with your Vercel domain
